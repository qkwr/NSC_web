# Auto Microphone Manual Test Checklist

ใช้ checklist นี้กับไมโครโฟนจริงเท่านั้น Fake microphone ใช้ตรวจ state/UI ได้ แต่ไม่ถือว่าเป็นหลักฐานว่าเสียงผู้ป่วยถูกบันทึกจริงแบบ end-to-end

## เงื่อนไขก่อนทดสอบ

- เปิดเว็บด้วย Chrome หรือ Edge ที่อนุญาตไมโครโฟน
- เปิด DevTools Console ใน development mode เพื่อดู log `[PoodPlearn auto recording]`
- ตรวจทั้ง `แบบฝึกเรียกชื่อภาพ` และ `แบบประเมินมาตรฐาน`
- หลังจบแต่ละข้อให้ดู DEV debug metadata: RMS, threshold, chunk count, Blob size, duration, stop reason

## กรณีทดสอบ

| # | กรณี | Expected result |
|---|---|---|
| 1 | ผู้ใช้พูดทันทีหลังเสียงอ่านโจทย์จบ | UI เปลี่ยนเป็นกำลังฟังทันทีหลัง TTS end, พยางค์แรกไม่หาย, ได้ Blob ขนาดมากกว่า 0, stopReason เป็น `silence` หรือ `manual-stop` |
| 2 | ผู้ใช้รอ 5-10 วินาทีก่อนพูด | ระบบยังรอฟังอยู่, เมื่อเริ่มพูด RMS ขึ้นเกิน threshold, ได้ Blob ขนาดมากกว่า 0 |
| 3 | ผู้ใช้หยุดคิด 1-2 วินาทีระหว่างคำ | ระบบยังไม่ตัดจบก่อนเวลา เพราะสั้นกว่า `endSilenceMs`, ได้ Blob ต่อเนื่อง |
| 4 | ผู้ใช้พูดเบา | RMS ต้องเห็นใน DEV debug; ถ้าระบบไม่จับเสียงให้บันทึกค่า RMS เพื่อ calibration threshold |
| 5 | ผู้ใช้พูดเสียงปกติ | จับเสียงได้, สถานะเปลี่ยนเป็นได้ยินแล้ว/กำลังบันทึก, ได้ Blob ขนาดมากกว่า 0 |
| 6 | มีเสียงพัดลมหรือเสียงพื้นหลัง | ไม่ควรเริ่มตัดสินว่าเป็นคำตอบจาก noise เบา ๆ; ถ้าจับผิดให้บันทึก RMS/noise floor เพื่อ calibration |
| 7 | ไม่มีการตอบ 15 วินาที | หยุดเป็น no-speech, ไม่ส่ง Blob ไปประมวลผล, ผู้ใช้ลองใหม่ได้ |
| 8 | พูดต่อเนื่องเกิน 30 วินาที | ระบบหยุดด้วย `max-duration`, สร้าง Blob ขนาดมากกว่า 0 และไม่ค้าง recording |
| 9 | กดหยุดตอบเอง | ระบบหยุดด้วย `manual-stop`, สร้าง Blob ขนาดมากกว่า 0 ถ้ามี audio chunk |
| 10 | เปลี่ยนข้อระหว่างกำลังฟัง/อ่านโจทย์ | callback เก่าต้องไม่เปิดไมค์ให้ข้อใหม่, ไม่มีเสียงโจทย์เก่าปน, ข้อใหม่เริ่ม flow ของตัวเอง |
| 11 | ออกจากหน้าระหว่างกำลังบันทึก | หยุด recorder, cancel TTS, release microphone stream, ไม่มี recording ค้าง |
| 12 | ปฏิเสธ permission ไมโครโฟน | เข้าสถานะ error พร้อมข้อความให้อนุญาตไมโครโฟน, ไม่เริ่ม TTS/recording ต่อแบบผิดจังหวะ |

## สิ่งที่ต้องบันทึกหลัง manual test

- Browser และ OS
- ไมโครโฟน/อุปกรณ์ที่ใช้
- MIME type ที่ console แสดง
- จำนวน chunk
- Blob size
- durationMs
- stopReason
- สรุปว่าเสียง TTS ปนเข้า Blob หรือไม่จากการฟังไฟล์/ตรวจฝั่ง ASR เมื่อมี service จริง
